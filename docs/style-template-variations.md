# テンプレート・AIプロンプト対応表

## 1. 目的

このドキュメントは、ラブリーゴーストライター風 Web アプリで使う「テンプレート」の候補を整理し、アプリ内で生成する予言文の方向性を管理するための設計資料である。

本アプリは非公式ファン作品であり、生成結果は娯楽用途の文章として扱う。文章は象徴的、不穏、暗示的な雰囲気を持たせるが、現実の危害、病気、死、犯罪、金銭損失などを断定的に予告しない。

## 2. 共通方針

### 2.1 予言文の共通制約

- 日本語で出力する。
- 1ヶ月分を4週に分ける。
- 各週は必ず1行にする。
- 入力値のうち、名前、相談テーマ、今月の気分を自然に反映する。
- 生年月日と性別は、必要な場合のみ象徴的に反映する。
- 直接的な不幸の予告ではなく、解釈の余地がある暗示に留める。
- 最終行は、読後に小さな行動や見直しへつながる余地を残す。

### 2.2 禁止または回避する表現

- 「必ず失敗する」「不幸になる」などの断定。
- 死、病気、事故、犯罪、破産などを直接予告する表現。
- 医療、法律、金融の専門判断を代替する助言。
- ユーザーを怖がらせるためだけの煽り。
- 公式素材、公式画像、既存キャラクター名への依存。

### 2.3 差し込み変数

テンプレートでは以下の変数を使う。

```text
{name}       ユーザー名
{theme}      相談テーマ
{mood}       今月の気分
{week}       週番号
{symbol}     象徴語
{place}      場所語
{object}     道具語
{action}     穏当な行動語
{toneHint}   行候補や語彙プロファイルごとのニュアンス
```

## 3. 行候補の選択方針

専用のスタイル分類IDは持たない。第1週の行候補を1つ選び、以降は直前の行候補が持つ `nextCandidates` から、次の週に使う行候補を貪欲に選ぶ。

候補が複数ある場合は、入力値、語彙プロファイル、行番号ごとの基本メタ情報、直前行との接続の自然さをスコア化し、最も高い候補を選ぶ。同点の場合のみ、入力値由来のシードと `randomSalt` で安定的にタイブレークする。

`nextCandidates` が空、未定義、または参照先が見つからない場合は、現在の週番号に対応する行候補からフォールバックする。ただしフォールバックでも、直前行の `profileId`、行番号ごとの基本メタ情報、相談テーマに近い候補だけを対象にし、全候補から無制限に選ばない。

## 4. テンプレートバリエーション

テンプレートは4行セットではなく、行番号ごとの候補カタログとして管理する。第1週の候補を選んだ後、その候補が持つ `nextCandidates` をたどりながら第2週以降の候補を選び、1ヶ月分の予言として組み合わせる。

`line1` から `line4` は、詩の中の固定役割ではなく週番号に対応する。

- `line1`: 第1週の候補。
- `line2`: 第2週の候補。
- `line3`: 第3週の候補。
- `line4`: 第4週の候補。

各週の候補文は、それぞれ1行だけで成立する予言文にする。名前、相談テーマ、今月の気分、小さな行動への余地は、特定の行だけに固定せず、各週の候補内へ必要に応じて自然に含める。

週番号は表示上の見出しで示し、詩本文には含めない。テンプレート本文にも `第1週`、`第{week}週` などの週番号表現は入れない。

実装上の構造は以下を基本にする。候補は `lineKey` ごとに分けて保持し、行候補の参照は `candidateId` で行う。

```ts
type TemplateLineKey = "line1" | "line2" | "line3" | "line4";

type TemplateLineCandidateRef = {
  lineKey: TemplateLineKey;
  candidateId: string;
};

type TemplateLineCandidateMeta = {
  reading: string;
  promptFocus: string;
  caution: string;
};

type TemplateLineCandidate = {
  candidateId: string;
  lineKey: TemplateLineKey;
  text: string;
  profileId: string;
  candidateMeta: TemplateLineCandidateMeta;
  nextCandidates?: TemplateLineCandidateRef[];
};

type TemplateLineCatalog = {
  line1: TemplateLineCandidate[];
  line2: TemplateLineCandidate[];
  line3: TemplateLineCandidate[];
  line4: TemplateLineCandidate[];
};
```

生成時は、第1週の候補を選んだあと、その候補が持つ `nextCandidates` から第2週の候補を貪欲に選ぶ。第2週、第3週も同じように、直前に選ばれた候補が次行候補を制約する。`nextCandidates` が空または未定義の場合のみ、現在の週番号に対応する行候補からフォールバック選択する。

たとえば、第1週で `line1` 候補 `T01-L1-A` が選ばれた場合、その候補の `nextCandidates` に `line2` 候補 `T10-L2-A` や `T17-L2-A` を持たせる。第2週はその候補群から選ぶため、単純な全体ランダムではなく、前行から自然につながる行だけが次に来る。

第1週は `T01-L1-A`、第2週は `T10-L2-A`、第3週は `T04-L3-A`、第4週は `T17-L4-A` のように組み合わせてよい。週番号と行番号の対応は固定し、`line1` の候補を第2週に使うような入れ替えはしない。

月全体の統一感は、`nextCandidates` に自然につながる行候補だけを列挙し、候補選択時に直前行との接続を重く評価することで保つ。各週の `candidateId` は `ProphecyWeek` に保持する。

### 4.1 行遷移の例

以下は構造を示すための例である。候補文は行ごとの配列に分かれ、次に来て自然な候補だけを `nextCandidates` に列挙する。

```ts
const templateLineCatalog: TemplateLineCatalog = {
  line1: [
    {
      candidateId: "T01-L1-A",
      lineKey: "line1",
      text: "{name}の名を薄いインクが覚え、{theme}の余白に小さな{symbol}が残る",
      profileId: "VP01",
      candidateMeta: {
        reading: "名前と相談テーマを、まだ書き換えられる紙面として導入する",
        promptFocus: "月初に観察する対象を一つ選ぶ",
        caution: "初手から結論や吉凶を決めない",
      },
      nextCandidates: [
        { lineKey: "line2", candidateId: "T01-L2-A" },
        { lineKey: "line2", candidateId: "T10-L2-A" },
        { lineKey: "line2", candidateId: "T17-L2-A" },
      ],
    },
  ],
  line2: [
    {
      candidateId: "T01-L2-A",
      lineKey: "line2",
      text: "{theme}へ向かう行間に小さな{object}が挟まれ、{name}は順番を{action}",
      profileId: "VP01",
      candidateMeta: {
        reading: "相談テーマの行間にある条件や順番を見直す",
        promptFocus: "前提、順序、境界を整理する",
        caution: "相手の反応や結果を断定しない",
      },
      nextCandidates: [
        { lineKey: "line3", candidateId: "T01-L3-A" },
        { lineKey: "line3", candidateId: "T12-L3-A" },
      ],
    },
  ],
  line3: [
    {
      candidateId: "T01-L3-A",
      lineKey: "line3",
      text: "{mood}の気配は古い活字の間で滲み、{theme}の読み方を少し変える",
      profileId: "VP17",
      candidateMeta: {
        reading: "今月の気分を、テーマの読み方を調整する合図として扱う",
        promptFocus: "感情から分かる調整点を拾う",
        caution: "不安を煽らず、感情を否定しない",
      },
      nextCandidates: [
        { lineKey: "line4", candidateId: "T01-L4-A" },
        { lineKey: "line4", candidateId: "T18-L4-A" },
      ],

  ],
  line4: [
    {
      candidateId: "T01-L4-A",
      lineKey: "line4",
      text: "最後の罫線を越える前に、{name}は手元の言葉を{action}とよい",
      profileId: "VP01",
      candidateMeta: {
        reading: "月末に向けて、手元で扱える言葉や行動へ着地する",

        caution: "成功保証や専門判断にしない",
      },
    },
  ],
};
```

`nextCandidates` は自然な接続を表す。たとえば、余白や紙面から始まる第1週は、時間管理へ急に飛ぶよりも、行間、保留、記録、見直しへ進む候補を多く持たせる。逆に、仕事や締切のテーマでタイプライター調の行が選ばれた場合は、次行候補に暦、砂時計、階段などを含めると流れが自然になる。

### 4.4 行別語彙プロファイル

テンプレート本文は詩としての骨格だけを持ち、各行に割り当てた語彙プロファイルから `{symbol}`、`{place}`、`{object}`、`{action}`、`{toneHint}`、`{adviceNuance}` を選ぶ。

同じ語彙プロファイルは複数の行候補に割り当ててよい。同じ語彙も複数プロファイルに含めてよい。たとえば「鍵」は、VP06では選択肢の象徴、VP15では現在地確認の道具、VP23では変わりにくい価値観の入口として使える。

```ts
type LineVocabularyProfile = {
  profileId: string;
  symbols: string[];
  places: string[];
  objects: string[];
  actions: string[];
  toneHints: string[];
  adviceNuances: string[];
  promptMeta: {
    focus: string;
    caution: string;
    actionFrame: string;
  };
};

type TemplateLineMeta = {
  lineKey: "line1" | "line2" | "line3" | "line4";
  weekNumber: 1 | 2 | 3 | 4;
  role: "導入" | "テーマ提示" | "気分の読み替え" | "行動への着地";
  promptQuestion: string;
  axis: string;
  inputUse: Array<"name" | "theme" | "mood" | "birthDate" | "gender">;
  promptFocus: string;
  caution: string;
  actionLanding: string;
};
```

#### 行メタ情報の組み立て

各行の `selectedTerms` には、語彙プロファイルから選ばれた語彙とメタ情報を保持する。

```ts
type SelectedLineTerms = {
  lineMeta: TemplateLineMeta;
  candidateMeta: TemplateLineCandidateMeta;
  profileId: string;
  symbol?: string;
  place?: string;
  object?: string;
  action?: string;
  toneHint?: string;
  adviceNuance?: string;
  promptMeta: {
    focus: string;
    caution: string;
    actionFrame: string;
    symbolMeta?: VocabularyPromptMeta;
    placeMeta?: VocabularyPromptMeta;
    objectMeta?: VocabularyPromptMeta;
    actionMeta?: VocabularyPromptMeta;
  };
};
```

`buildInterpretationAxis` では、各行について以下の順で解釈軸を作る。

1. `lineMeta.role` と `lineMeta.axis` で週の役割と解釈の方向を決める。
2. `lineMeta.inputUse` を見て、入力値のうち自然に反映する要素を決める。
3. `candidateMeta.reading` で候補文単体の読みを補う。
4. `focus`、`lineMeta.promptFocus`、`candidateMeta.promptFocus` で助言の焦点を決める。
5. `symbolMeta`、`placeMeta`、`objectMeta` で詩の根拠語を拾う。
6. `actionMeta`、`actionFrame`、`lineMeta.actionLanding` で、現実的な小さい行動へ落とす。
7. `caution`、`lineMeta.caution`、`candidateMeta.caution` を使い、断定や煽りを避ける。

## 6. テンプレート・AIプロンプト対応表

AI用プロンプトは、予言文をそのまま拡張するのではなく、相談テーマに関係する現実的な助言へ読み替える。解釈軸は、固定の分類名や分類IDではなく、実際に選ばれた4行、各行の `candidateId`、`profileId`、行番号ごとの基本メタ情報、差し込まれた語彙から作る。

### 6.1 解釈軸の生成ルール

`四行詩の解釈軸` は、以下を週ごとに結合して作る。

```text
第{weekNumber}週は、{lineMeta.role}として読む。
候補文は、{candidateMeta.reading}として扱う。
詩中の「{selected symbol/place/object}」は、{vocabularyMeta.reading}の比喩として扱う。
助言では{lineProfile.promptMeta.focus}、{lineMeta.promptFocus}、{candidateMeta.promptFocus}を中心にする。
注意点は{lineProfile.promptMeta.caution}、{lineMeta.caution}、{candidateMeta.caution}に従う。
行動は「{selected action}」のメタ情報に沿って、{lineMeta.actionLanding}程度の小さな一手へ落とす。
```

月全体の解釈軸は、選ばれた4行の `promptMeta.focus`、`lineMeta.axis`、`actionLanding` をつないで作る。

```text
全体として、{week1.axis}から始まり、{week2.axis}で条件を見直し、{week3.axis}で気分を読み替え、{week4.axis}で小さな行動へ着地する流れとして読む。
相談テーマ「{theme}」については、実際に選ばれた語彙と行候補だけを根拠に解釈する。
```

### 6.2 メタ情報の優先順位

同じ語彙が複数の行候補や複数プロファイルに出る場合は、以下の優先順位で意味を決める。

1. その行に割り当てられた `LineVocabularyProfile.promptMeta`
2. 候補単位の `TemplateLineCandidateMeta`
3. 行番号ごとの `TemplateLineMeta`
4. 選ばれた語彙の `VocabularyPromptMeta`
5. ユーザー入力の `theme` と `mood`

たとえば「鍵」が選ばれた場合でも、VP06 では「選択条件」、VP15 では「現在地確認」、VP23 では「変えない部分と変える部分の境界」として扱う。語彙そのものの意味を固定しすぎず、行プロファイルで読み替える。

## 7. AIプロンプト雛形

AI用プロンプトは相談テーマごとに分けず、以下の共通プロンプトを1つだけ使う。
生成された四行詩を読む観点は、実際に選ばれた4行、各行の `candidateId` と `profileId`、差し込まれた象徴語・道具語・行動語、週ごとの助言ニュアンスから組み立てた `{interpretationAxis}` に差し込む。

```text
あなたは日本語で、占い風の四行詩を現実的な助言に読み替えるアシスタントです。
以下のユーザー入力と四行詩を読み取り、相談テーマに関係する助言を返してください。

【ユーザー入力】
名前: {name}
生年月日: {birthDate}
性別: {gender}
相談テーマ: {theme}
今月の気分: {mood}

【四行詩】
{weeks}

【四行詩の解釈軸】
{interpretationAxis}

【出力条件】
- 日本語で出力する
- 四行詩の内容を助言の根拠として扱う
- 四行詩の解釈軸を、読み替えの観点として扱う
- ユーザーの相談テーマに直接関係する助言を返す
- 不穏さを煽らず、次に取れる小さな行動へ落とし込む
- 占いや予言として断定せず、解釈の一例として提示する
- 医療、法律、金融など専門判断が必要な領域では、専門家への相談を促す
- 相手の気持ち、未来の出来事、成功失敗を断定しない

【出力形式】
1. 四行詩から読める今月の流れ
2. 相談テーマへの解釈
3. 今週からできる小さな行動
4. 注意したい思い込み
```

## 8. 実装時の選択ロジック案

### 8.1 行候補選択

入力値由来のシードと再生成用の `randomSalt` を組み合わせ、`line1` の行候補から第1週の候補を1つ選ぶ。第2週以降は、直前に選ばれた行候補の `nextCandidates` に含まれる候補から貪欲に選ぶ。

たとえば `week1Candidate = T01-L1-A` が選ばれ、その候補の `nextCandidates` に `T10-L2-A` と `T17-L2-A` がある場合、第2週はその2候補から選ぶ。第2週で選ばれた候補がさらに第3週候補を持ち、第3週が第4週候補を持つ。

`nextCandidates` は、語彙プロファイル、相談テーマとの相性、週の役割、直前行との接続の自然さをもとに用意する。第1週から第4週まで完全に独立したランダム選択にはしない。

候補が複数ある場合は、以下の観点をスコア化し、最も高い候補を選ぶ。

- 直前行の `profileId` と自然につながるか。
- 直前行の `lineMeta.axis` から、次の週の `lineMeta.axis` へ流れがあるか。
- `theme` と `mood` に含まれる語と、候補の語彙プロファイルが近いか。
- 同じ `profileId` に閉じすぎず、月全体の流れに変化があるか。
- 禁止または回避する表現に近づかないか。

同点の場合のみ、入力値由来のシードと `randomSalt` で安定的にタイブレークする。

`nextCandidates` が空、未定義、または参照先が見つからない場合は、現在の週番号に対応する行だけをフォールバック候補にする。フォールバックでも全行候補から無制限に選ばず、直前行の `profileId`、行番号ごとの基本メタ情報、相談テーマに近い候補だけを対象にする。

```text
line1Candidates -> week1 seededRandom selection
week1Candidate.nextCandidates -> week2 greedy selection
week2Candidate.nextCandidates -> week3 greedy selection
week3Candidate.nextCandidates -> week4 greedy selection
```

### 8.2 AIプロンプト生成

AI用プロンプトは、相談テーマに関係なく `7. AIプロンプト雛形` の共通プロンプトを使う。
相談テーマによるプロンプト雛形の出し分けは行わない。`四行詩の解釈軸` は、対応表の基礎語彙だけで固定せず、生成時に選ばれた4行、各行の `candidateId` と `profileId`、差し込まれた象徴語・道具語・行動語、週ごとの助言ニュアンスを集約して作る。

```text
selectedWeeks -> selectedTerms -> weeklyNuances -> interpretationAxis
```

解釈軸に含める内容:

- 各週の詩から読み取る観点
- 実際に差し込まれた象徴語、場所語、道具語、行動語
- 週ごとの助言ニュアンス
- 4週全体を通した流れ
- ユーザーの相談テーマと今月の気分への接続

解釈軸に含めない内容:

- 実際に選ばれていない語彙
- 未来、相手の気持ち、成功失敗の断定

## 9. 生成例

### 9.1 入力例

```text
名前: 灯
相談テーマ: 今月の仕事の進め方
今月の気分: 少し不安がある
```

### 9.2 生成結果

```text
灯の紙面に短い音が落ちる
今月の仕事の進め方は一度だけ行頭へ戻される
少し不安がある文字は、強く打つほど少しにじむ
間隔を空けて、見直すことが鍵になる
```

### 9.3 AIプロンプト差し込み

```text
四行詩の解釈軸: 第1週は短い音から作業の始点を確かめる週、第2週は行頭へ戻る表現から進め方を一度見直す週、第3週は不安が強いほど文字がにじむ流れから力の入れ方を調整する週、第4週は間隔を空けて見直す行動を鍵として扱う週として読む。全体として、今月の仕事の進め方を急いで確定せず、短い単位に分けて確認し直す流れとして解釈する。
使用プロンプト: 共通プロンプト
```

## 10. 実装メモ

- 候補 ID、行番号ごとの基本メタ情報、解釈軸の基礎語彙は定数化する。最終的な四行詩の解釈軸は、実際に選ばれた4行と語彙から生成する。
- 禁止語彙は生成結果に含まれないよう、テンプレート側で避ける。
- AI用プロンプトには、実際に選ばれた4行と語彙から組み立てた四行詩の解釈軸を含めると、助言の方向が安定する。
- 4週分は、直前の行候補の `nextCandidates` をたどって次行を貪欲に選ぶ。第1週は T01 の `line1`、第2週は T10 の `line2` のような混合は可能だが、次行候補として明示された自然な接続だけを基本にする。
- 再生成では `randomSalt` により第1週候補や語彙を変えつつ、相談テーマから大きく外れないようにする。
