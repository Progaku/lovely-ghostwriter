# 文体・テンプレート・AIプロンプト対応表

## 1. 目的

このドキュメントは、ラブリーゴーストライター風 Web アプリで使う「文体」と「テンプレート」の候補を整理し、アプリ内で生成する予言文の方向性を管理するための設計資料である。

本アプリは非公式ファン作品であり、生成結果は娯楽用途の文章として扱う。文体は象徴的、不穏、暗示的な雰囲気を持たせるが、現実の危害、病気、死、犯罪、金銭損失などを断定的に予告しない。

## 2. 共通方針

### 2.1 予言文の共通制約

- 日本語で出力する。
- 1ヶ月分を4週に分ける。
- 各週は必ず1行にする。
- 入力値のうち、名前、相談テーマ、今月の気分を自然に反映する。
- 生年月日と性別は、必要な場合のみ象徴的に反映する。
- 直接的な不幸の予告ではなく、解釈の余地がある暗示に留める。
- 最終行は、読後に小さな行動や見直しへつながる余地を残す。

### 2.2 禁止または回避する表現

- 「必ず失敗する」「不幸になる」などの断定。
- 死、病気、事故、犯罪、破産などを直接予告する表現。
- 医療、法律、金融の専門判断を代替する助言。
- ユーザーを怖がらせるためだけの煽り。
- 公式素材、公式画像、既存キャラクター名への依存。

### 2.3 差し込み変数

テンプレートでは以下の変数を使う。

```text
{name}       ユーザー名
{theme}      相談テーマ
{mood}       今月の気分
{week}       週番号
{symbol}     象徴語
{place}      場所語
{object}     道具語
{action}     穏当な行動語
{toneHint}   文体ごとのニュアンス
```

## 4. テンプレートバリエーション

各テンプレートは4週分で構成する。ただし、テンプレートバリエーションを4行セットとしてそのまま使わない。文体 ID とシード付き乱数により、週ごとにテンプレート ID と行候補を個別に選び、1ヶ月分の予言として組み合わせる。

`line1` から `line4` は、詩の中の役割ではなく週番号に対応する。

- `line1`: 第1週の候補。
- `line2`: 第2週の候補。
- `line3`: 第3週の候補。
- `line4`: 第4週の候補。

各週の候補文は、それぞれ1行だけで成立する予言文にする。名前、相談テーマ、今月の気分、小さな行動への余地は、特定の行だけに固定せず、各週の候補内へ必要に応じて自然に含める。

週番号は表示上の見出しで示し、詩本文には含めない。テンプレート本文にも `第1週`、`第{week}週` などの週番号表現は入れない。

実装上の構造は以下を基本にする。

```ts
type TemplateVariation = {
  templateId: string;
  styleIds: string[];
  line1: string[];
  line2: string[];
  line3: string[];
  line4: string[];
};
```

生成時は、まず第1週から第4週までそれぞれにテンプレート ID を選ぶ。次に、第1週なら選ばれたテンプレートの `line1` から1文、第2週なら選ばれたテンプレートの `line2` から1文、第3週なら選ばれたテンプレートの `line3` から1文、第4週なら選ばれたテンプレートの `line4` から1文を選ぶ。

たとえば、第1週は T01 の `line1`、第2週は T10 の `line2`、第3週は T04 の `line3`、第4週は T17 の `line4` のように組み合わせてよい。週番号と行番号の対応は固定し、T10 の `line1` を第2週に使うような入れ替えはしない。

文体の統一感は、最初に選んだ文体 ID に対応するテンプレート候補から週ごとのテンプレート ID を選ぶことで保つ。必要に応じて、相談テーマに合う近接文体のテンプレートを少量混ぜてもよいが、各週の `templateId` は `ProphecyWeek` に保持する。

### 週別候補の例

```ts
const paperTemplate: TemplateVariation = {
  templateId: "T01",
  styleIds: ["S01", "S07", "S17"],
  line1: [
    "{name}の名を薄いインクが覚え、{theme}の余白に小さな{symbol}が残る",
    "紙面の隅で{name}の文字が乾き、{mood}の影が少しだけ向きを変える",
    "{theme}の欄に細い罫線が引かれ、急がず{action}印が見える",
  ],
  line2: [
    "{theme}へ向かう行間に小さな{object}が挟まれ、{name}は順番を{action}",
    "{mood}という余白は読まれる前に深くなり、{symbol}だけが静かに光る",
    "{theme}という見出しはまだ余白を残し、ひとつ{action}余地を置く",
  ],
  line3: [
    "{mood}の気配は古い活字の間で滲み、{theme}の読み方を少し変える",
    "{name}の手元で{object}が黙り、見えたことを{action}時が来る",
    "{symbol}の影が紙の端で揺れ、{theme}は別の置き場所を求める",
  ],
  line4: [
    "最後の罫線を越える前に、{name}は手元の言葉を{action}とよい",
    "{theme}の余白に残る印を読み、次へ移る前にひとつ{action}",
    "{mood}の影が薄くなるころ、急がず{action}ことで月が閉じる",
  ],
};
```


`buildInterpretationAxis` では、各行について以下の順で解釈軸を作る。

1. `lineMeta.role` で週の役割を決める。
2. `focus` で助言の焦点を決める。
3. `symbolMeta`、`placeMeta`、`objectMeta` で詩の根拠語を拾う。
4. `actionMeta` と `actionFrame` で、現実的な小さい行動へ落とす。
5. `caution` を使い、断定や煽りを避ける。

## 5. 語彙バリエーション

語彙は単なる差し込み語ではなく、AI用プロンプトの `四行詩の解釈軸` を作るためのメタ情報を持つ。

```ts
type VocabularyPromptMeta = {
  word: string;
  category: "symbol" | "place" | "object" | "action";
  reading: string;
  promptFocus: string;
  caution: string;
  actionHint?: string;
};
```

### 5.1 象徴語

| 語彙 | reading | promptFocus | caution | actionHint |
| --- | --- | --- | --- | --- |

| S29 | T29 | VP29, VP14 | 種や枝を手入れし、増やすより育てる対象を絞る流れ | 長期計画、創作、習慣 | 医療的効能を示唆しない | 手入れする対象を一つ選ぶ |
| S30 | T30 | VP30, VP22 | 仮止めから始め、関係や作業を微修正する流れ | 関係修復、文章、仕事の調整 | 怪我や痛みの直接表現を避ける | 確定前に試す表現を作る |
| S31 | T31 | VP31, VP10 | 砂の粒を小さく分け、焦りを優先順位へ読み替える流れ | 締切、優先順位、時間配分 | 焦りや時間切れを煽らない | 今日の粒だけ扱う |
| S32 | T32 | VP32, VP24 | 雑音から必要な声を選び、情報との距離を調整する流れ | 情報過多、噂、連絡 | 陰謀めいた断定を避ける | 受け取る情報を一つ減らす |

### 6.1 解釈軸の生成ルール

`四行詩の解釈軸` は、文体IDの説明だけで作らない。実装では以下を結合して週ごとの読み替えを作る。

```text
第{weekNumber}週は、{lineMeta.role}として読む。
詩中の「{selected symbol/place/object}」は、{vocabularyMeta.reading}の比喩として扱う。
助言では{lineProfile.promptMeta.focus}を中心にし、{lineProfile.promptMeta.caution}に注意する。
行動は「{selected action}」のメタ情報に沿って、{actionMeta.actionHint}程度の小さな一手へ落とす。
```

月全体の解釈軸は、選ばれた4行の `promptMeta.focus` と `monthlyFlow` をまとめて作る。

```text
全体として、{stylePromptMeta.monthlyFlow}として読む。
相談テーマ「{theme}」については、{stylePromptMeta.promptFocus}を中心に解釈し、{stylePromptMeta.promptCaution}。
次の行動は、{stylePromptMeta.outputActionBias}程度に留める。
```

### 6.2 メタ情報の優先順位

同じ語彙が複数テンプレートや複数プロファイルに出る場合は、以下の優先順位で意味を決める。

1. その行に割り当てられた `LineVocabularyProfile.promptMeta`
2. 選ばれた語彙の `VocabularyPromptMeta`
3. 文体の `StylePromptMeta`
4. 行番号の `TemplateLineMeta`
5. ユーザー入力の `theme` と `mood`

たとえば「鍵」が選ばれた場合でも、T06 / VP06 では「選択条件」、T15 / VP15 では「現在地確認」、T23 / VP23 では「変えない部分と変える部分の境界」として扱う。語彙そのものの意味を固定しすぎず、行プロファイルで読み替える。

## 7. AIプロンプト雛形

AI用プロンプトは文体や相談テーマごとに分けず、以下の共通プロンプトを1つだけ使う。
生成された四行詩を読む観点は、実際に選ばれた4行、各行のテンプレート、差し込まれた象徴語・道具語・行動語、週ごとの助言ニュアンスから組み立てた `{interpretationAxis}` に差し込む。

```text
あなたは日本語で、占い風の四行詩を現実的な助言に読み替えるアシスタントです。
以下のユーザー入力と四行詩を読み取り、相談テーマに関係する助言を返してください。

【ユーザー入力】
名前: {name}
生年月日: {birthDate}
性別: {gender}
相談テーマ: {theme}
今月の気分: {mood}

【四行詩】
{weeks}

【四行詩の解釈軸】
{interpretationAxis}

【出力条件】
- 日本語で出力する
- 四行詩の内容を助言の根拠として扱う
- 四行詩の解釈軸を、読み替えの観点として扱う
- ユーザーの相談テーマに直接関係する助言を返す
- 不穏さを煽らず、次に取れる小さな行動へ落とし込む
- 占いや予言として断定せず、解釈の一例として提示する
- 医療、法律、金融など専門判断が必要な領域では、専門家への相談を促す
- 相手の気持ち、未来の出来事、成功失敗を断定しない

【出力形式】
1. 四行詩から読める今月の流れ
2. 相談テーマへの解釈
3. 今週からできる小さな行動
4. 注意したい思い込み
```

## 8. 実装時の選択ロジック案

### 8.1 文体選択

入力値由来のシードと再生成用の `randomSalt` を組み合わせ、文体を1つ選ぶ。相談テーマに含まれる語から候補を絞ってもよい。

| 入力傾向 | 優先文体 |
| --- | --- |
| 恋愛、連絡、相手、友人 | S04, S05, S11, S22, S28 |
| 仕事、勉強、締切、転職 | S03, S06, S10, S19, S31 |
| 不安、疲れ、迷い | S08, S17, S18, S21, S26 |
| 創作、習慣、生活 | S07, S14, S24, S29, S30 |
| 将来、進路、選択 | S06, S09, S15, S23, S25 |

### 8.2 テンプレート選択

文体 ID に紐づくテンプレート候補から、週ごとにテンプレート ID を1つずつ選ぶ。次に、その週のテンプレート ID と行番号に対応する候補文を選ぶ。

たとえば `week1TemplateId = T01`、`week2TemplateId = T10`、`week3TemplateId = T04`、`week4TemplateId = T17` のように選び、第1週は T01 の `line1`、第2週は T10 の `line2`、第3週は T04 の `line3`、第4週は T17 の `line4` を使う。

1ヶ月全体で文体の統一感を保つため、基本的には選ばれた文体 ID に対応するテンプレート候補内で週ごとのテンプレート ID を選ぶ。同じ文体に対応するテンプレートが少ない場合や、相談テーマとの相性を上げたい場合は、近接文体のテンプレートを混ぜてもよい。ただし、週番号の対応は守り、第1週候補を第3週に使うような入れ替えはしない。

```text
styleId -> candidateTemplateIds
candidateTemplateIds -> week1TemplateId -> line1Candidates -> week1 seededRandom selection
candidateTemplateIds -> week2TemplateId -> line2Candidates -> week2 seededRandom selection
candidateTemplateIds -> week3TemplateId -> line3Candidates -> week3 seededRandom selection
candidateTemplateIds -> week4TemplateId -> line4Candidates -> week4 seededRandom selection
```

### 8.3 AIプロンプト生成

AI用プロンプトは、相談テーマや文体に関係なく `7. AIプロンプト雛形` の共通プロンプトを使う。
相談テーマによるプロンプト雛形の出し分けは行わない。`四行詩の解釈軸` は、対応表の基礎語彙だけで固定せず、生成時に選ばれた4行、各行のテンプレート、差し込まれた象徴語・道具語・行動語、週ごとの助言ニュアンスを集約して作る。

```text
selectedWeeks -> selectedTemplateIds -> selectedTerms -> weeklyNuances -> interpretationAxis
```

解釈軸に含める内容:

- 各週の詩から読み取る観点
- 実際に差し込まれた象徴語、場所語、道具語、行動語
- 週ごとの助言ニュアンス
- 4週全体を通した流れ
- ユーザーの相談テーマと今月の気分への接続

解釈軸に含めない内容:

- AI用プロンプト上の文体名
- 実際に選ばれていない語彙
- 未来、相手の気持ち、成功失敗の断定

## 9. 生成例

### 9.1 入力例

```text
名前: 灯
相談テーマ: 今月の仕事の進め方
今月の気分: 少し不安がある
```

### 9.2 文体 S03 / テンプレート T03

```text
灯の紙面に短い音が落ちる
今月の仕事の進め方は一度だけ行頭へ戻される
少し不安がある文字は、強く打つほど少しにじむ
間隔を空けて、見直すことが鍵になる
```

### 9.3 AIプロンプト差し込み

```text
四行詩の解釈軸: 第1週は短い音から作業の始点を確かめる週、第2週は行頭へ戻る表現から進め方を一度見直す週、第3週は不安が強いほど文字がにじむ流れから力の入れ方を調整する週、第4週は間隔を空けて見直す行動を鍵として扱う週として読む。全体として、今月の仕事の進め方を急いで確定せず、短い単位に分けて確認し直す流れとして解釈する。
使用プロンプト: 共通プロンプト
```

## 10. 実装メモ

- 文体 ID、テンプレート ID、解釈軸の基礎語彙は定数化する。最終的な四行詩の解釈軸は、実際に選ばれた4行と語彙から生成する。
- 禁止語彙は生成結果に含まれないよう、テンプレート側で避ける。
- AI用プロンプトには文体名を含めず、実際に選ばれた4行と語彙から組み立てた四行詩の解釈軸を含めると、助言の方向が安定する。
- 4週分は同じ文体 ID を軸にしつつ、週ごとにテンプレート ID を選び直す。第1週は T01 の `line1`、第2週は T10 の `line2` のような混合を基本にすると、固定の4行セット感が薄れる。
- 再生成では `randomSalt` により文体や語彙を変えつつ、相談テーマから大きく外れないようにする。
