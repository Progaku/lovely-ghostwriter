# ラブリーゴーストライター風 Webアプリ 実装仕様

## 1. 目的

`./requirements.md` をもとに、React/Vite/TypeScript で初期実装するための仕様を定義する。

本仕様は、フロントエンド単体SPAとして動作する初期版を対象にする。バックエンド、DB、認証、外部AI API連携、永続保存、共有機能は含めない。

## 2. 実装スコープ

### 2.1 実装するもの

- 日本語UIの単一ページアプリ
- 入力フォーム
- 入力バリデーション
- 4週分、各週4行の予言テキスト生成
- 再生成
- AI用プロンプト生成
- AI用プロンプトのコピー
- コピー結果のフィードバック表示
- 非公式ファン作品、娯楽用途、重要判断非推奨の注意書き
- スマートフォンとデスクトップに対応したレスポンシブ表示

### 2.2 実装しないもの

- 外部AI API呼び出し
- サーバーサイド処理
- データベース保存
- 履歴表示
- URL共有
- 画像保存
- SNS投稿
- ユーザーアカウント
- 多言語対応
- 公式画像、漫画コマ、アニメ素材、既存キャラクター画像の使用

## 3. 技術仕様

### 3.1 技術スタック

- React
- Vite
- TypeScript
- CSS Modules または通常のCSS

初期実装では状態管理ライブラリ、ルーティングライブラリ、UIコンポーネントライブラリは必須にしない。単一画面のため、React標準の `useState` と純粋関数で実装する。

### 3.2 推奨ディレクトリ構成

```text
src/
  App.tsx
  main.tsx
  styles.css
  components/
    ProphecyForm.tsx
    ProphecyResultView.tsx
    AiPromptPanel.tsx
    Notice.tsx

    promptBuilder.ts
    validation.ts
  types/
    prophecy.ts
```

初期実装ではファイル数を増やしすぎず、必要に応じて `App.tsx` から分割する。

## 4. データ型

### 4.1 入力データ

```ts
export type Gender = "男性" | "女性" | "その他" | "";

export type ProphecyInput = {
  name: string;
  birthDate: string;
  gender: Gender;
  theme: string;
  mood: string;
};
```

### 4.2 生成結果

```ts
export type ProphecyWeek = {
  weekNumber: 1 | 2 | 3 | 4;
  lines: [string, string, string, string];
};

export type ProphecyResult = {
  weeks: ProphecyWeek[];
  aiPrompt: string;
  generatedAt: string;
};
```

`generatedAt` は画面表示必須ではないが、再生成やテスト時に結果生成のタイミングを確認できるよう保持する。

### 4.3 バリデーションエラー

```ts
export type ValidationErrors = Partial<Record<keyof ProphecyInput, string>>;
```

## 5. 画面仕様

### 5.1 画面全体

単一ページ構成にする。

表示順は以下とする。

1. タイトル領域
2. 注意書き
3. 入力フォーム
4. 生成結果
5. AI用プロンプト

結果が未生成の場合、4と5は表示しない。

### 5.2 タイトル領域

以下を表示する。

- アプリ名
- 短い補足文

文言例:

- アプリ名: `Lovely Ghostwriter`
- 補足文: `4週分の四行詩をつくる非公式ファンWebアプリ`

公式作品と誤認される表現は避ける。

### 5.3 注意書き

画面上部またはフォーム付近に、最小限の注意書きを表示する。

必須文言:

```text
このアプリは非公式ファン作品です。生成結果は娯楽用途の文章であり、重要な判断には使用しないでください。
```

### 5.4 入力フォーム

#### 5.4.1 項目

| 項目 | UI | 必須 | 上限 | 備考 |
| --- | --- | --- | --- | --- |
| 名前 | テキスト入力 | 必須 | 50文字 | 前後空白はバリデーション時に除去 |
| 生年月日 | 日付入力 | 必須 | - | `input type="date"` を使う |
| 性別 | セレクト | 任意 | - | 未選択、男性、女性、その他 |
| 相談テーマ | テキストエリア | 必須 | 255文字 | 例: 仕事、人間関係、創作など |
| 今月の気分 | セレクト + 自由入力、またはテキスト入力 | 必須 | 255文字 | 初期版はテキスト入力でも可 |

#### 5.4.2 入力補助

- 各入力には日本語のラベルを付ける。
- 必須項目は視覚的に分かるようにする。
- 上限のある項目は `maxLength` を設定する。
- テキストエリアは長文でもレイアウトが崩れない幅にする。

#### 5.4.3 操作ボタン

初回生成前:

- `予言を生成する`

生成後:

- `再生成する`

ボタン押下時、必須項目にエラーがある場合は生成しない。

### 5.5 結果表示

生成後に、4週分を週ごとに表示する。

各週の表示要素:

- `第1週` から `第4週` の見出し
- 4行の詩

改行を保持して読みやすく表示する。各行は個別の `<p>` または `<li>` として出力してよい。

### 5.6 AI用プロンプト表示

生成後に、AI用プロンプトを表示する。

表示要素:

- 見出し
- コピーボタン
- プロンプト本文
- コピー成功または失敗の短いフィードバック

プロンプト本文は手動コピーできるように、`textarea` または選択可能なテキスト領域で表示する。

## 6. 状態管理仕様

`App.tsx` で以下の状態を管理する。

```ts
const [input, setInput] = useState<ProphecyInput>(initialInput);
const [errors, setErrors] = useState<ValidationErrors>({});
const [result, setResult] = useState<ProphecyResult | null>(null);
const [copyStatus, setCopyStatus] = useState<"idle" | "success" | "error">("idle");
```

### 6.1 状態遷移

#### 入力変更

- 対象項目の値を更新する。
- コピー状態は変更しない。
- 既存結果は自動では消さない。

#### 生成ボタン押下

1. 入力値をバリデーションする。
2. エラーがあれば `errors` を更新し、`result` は変更しない。
3. エラーがなければ予言を生成する。
4. 生成した予言からAI用プロンプトを生成する。
5. `result` を更新する。
6. `errors` を空にする。
7. `copyStatus` を `idle` に戻す。

#### 再生成ボタン押下

生成ボタン押下と同じ処理を行う。同じ入力でも表現が変わるよう、生成ロジック内で乱数を使用する。

#### コピーボタン押下

1. `navigator.clipboard.writeText(result.aiPrompt)` を呼ぶ。
2. 成功時は `copyStatus` を `success` にする。
3. 失敗時は `copyStatus` を `error` にする。
4. 失敗時もプロンプト本文は選択可能なままにする。

## 7. バリデーション仕様

### 7.1 共通ルール

- 必須項目は `trim()` 後に空文字ならエラー。
- 上限文字数を超える入力は、HTML属性とバリデーション関数の両方で防ぐ。
- エラーメッセージは項目付近に表示する。
- エラーがある状態では生成処理を実行しない。

### 7.2 エラーメッセージ

| 項目 | 条件 | メッセージ |
| --- | --- | --- |
| 名前 | 空 | 名前を入力してください。 |
| 名前 | 51文字以上 | 名前は50文字以内で入力してください。 |
| 生年月日 | 空 | 生年月日を入力してください。 |
| 相談テーマ | 空 | 相談テーマを入力してください。 |
| 相談テーマ | 256文字以上 | 相談テーマは255文字以内で入力してください。 |
| 今月の気分 | 空 | 今月の気分を入力してください。 |
| 今月の気分 | 256文字以上 | 今月の気分は255文字以内で入力してください。 |

## 8. 予言生成仕様

### 8.1 生成方式

外部AI APIは使わず、アプリ内のテンプレートと語彙リストを組み合わせて生成する。

生成関数:

```ts
export function generateProphecy(input: ProphecyInput): ProphecyWeek[];
```

### 8.2 出力制約

- 常に4週分を返す。
- `weekNumber` は1から4まで順に設定する。
- 各週の `lines` は必ず4要素のタプルにする。
- 各行は日本語文字列にする。
- 空行は生成しない。
- 危害、病気、死、犯罪、金銭損失を断定的に予告する語句は使わない。

### 8.3 文体

象徴的、不穏、暗示的な雰囲気にする。ただし、現実の被害や重大な不幸を断定しない。

推奨する語彙:

- 紙
- インク
- 窓
- 鍵
- 月
- 影
- 針
- 机
- 封筒
- 静けさ
- 足音
- 余白

避ける語彙:

- 死ぬ
- 殺される
- 病気になる
- 破産する
- 犯罪に巻き込まれる
- 事故に遭う
- 必ず失敗する

### 8.4 テンプレート例

以下のような複数テンプレートからランダム選択する。

```ts
const lineTemplates = [
  "{name}の名を、薄いインクがまだ覚えている",
  "{theme}へ向かう道に、小さな鍵が落ちている",
  "{mood}の影は、窓辺で形を変える",
  "急がず拾えば、余白に次の印が残る",
];
```

実装時は、週番号や入力値を反映できるテンプレートを複数用意する。

### 8.5 ランダム性

再生成で多少異なる結果にするため、以下のいずれかをランダム化する。

- テンプレート選択
- 象徴語の選択
- 行末表現
- 週ごとの助言ニュアンス

暗号学的な乱数は不要。`Math.random()` でよい。

### 8.6 入力値の扱い

- `name`、`theme`、`mood` は `trim()` した値を使用する。
- `birthDate` は予言本文に必ず含めなくてよい。
- `gender` は未選択なら本文に含めなくてよい。
- 入力値はHTMLとして描画せず、Reactの通常のテキスト描画に任せる。

## 9. AI用プロンプト生成仕様

### 9.1 生成関数

```ts
export function buildAiPrompt(input: ProphecyInput, weeks: ProphecyWeek[]): string;
```

### 9.2 プロンプト構成

AI用プロンプトは以下の順に組み立てる。

1. 依頼文
2. ユーザー入力
3. アプリ内生成の四行詩
4. 出力条件
5. 注意事項

### 9.3 プロンプト文面

以下の内容を含める。

```text
あなたは日本語で、占い風テキストを現実的な助言に読み替えるアシスタントです。
以下のユーザー入力と四行詩を読み取り、相談テーマに関係する助言を返してください。

【ユーザー入力】
名前:
生年月日:
性別:
相談テーマ:
今月の気分:

【四行詩】
第1週
...

【出力条件】
- 日本語で出力する
- 四行詩の内容を助言の根拠として扱う
- ユーザーの相談テーマに直接関係する助言を返す
- 不穏さを煽らず、次に取れる小さな行動へ落とし込む
- 占いや予言として断定せず、解釈の一例として提示する
- 医療、法律、金融など専門判断が必要な領域では、専門家への相談を促す
```

### 9.4 プロンプトの表示

- 改行を保持する。
- ユーザーが全文を選択できる。
- コピーに失敗しても本文が読める。

## 10. コピー機能仕様

### 10.1 成功時

表示文言:

```text
コピーしました。
```

### 10.2 失敗時

表示文言:

```text
コピーできませんでした。本文を選択して手動でコピーしてください。
```

### 10.3 実装上の注意

- Clipboard APIが使えない環境では失敗扱いにする。
- 失敗時にアプリがクラッシュしないよう `try/catch` で処理する。

## 11. デザイン仕様

### 11.1 方向性

古い占い紙面、紙、インク、タイプライター、古い机上、自動筆記を連想する静かなUIにする。

### 11.2 禁止事項

- 公式画像を使わない。
- 漫画コマやアニメ素材を使わない。
- 既存キャラクター画像を使わない。
- 読みにくいほど暗い配色にしない。
- ボタンや入力欄のテキストがはみ出すレイアウトにしない。

### 11.3 レイアウト

- 最大幅は `960px` 前後に制限する。
- スマートフォンでは1カラムにする。
- デスクトップではフォームと結果を適度な余白で並べるか、読み順優先の1カラムにする。
- 入力欄、ボタン、プロンプト本文は幅100%を基本にする。
- 長い相談テーマやプロンプトで横スクロールが出ないようにする。

### 11.4 色と質感

推奨:

- 背景: 暗い木机または黒に近い茶色
- メイン面: 古紙風の淡い紙色
- 文字: 濃いインク色
- アクセント: 深い赤、鈍い金、くすんだ青など少量

CSSだけで紙の質感を表現してよい。画像アセットは必須ではない。

## 12. アクセシビリティ仕様

- 入力項目には `label` を関連付ける。
- エラーメッセージは項目の近くに表示する。
- ボタンはキーボードで操作できる。
- 結果生成後もページ全体が読みやすいコントラストを保つ。
- コピー結果の文言はテキストで表示する。
- 色だけで必須やエラーを伝えない。

## 13. エラー処理

### 13.1 入力エラー

バリデーションメッセージを表示し、生成しない。

### 13.2 コピーエラー

コピー失敗メッセージを表示し、手動コピー可能な本文を維持する。

### 13.3 生成エラー

テンプレート生成はローカル処理のため原則失敗しない設計にする。万一例外が発生する実装にする場合は、画面に以下を表示する。

```text
生成中に問題が発生しました。入力内容を確認して、もう一度お試しください。
```

## 14. テスト仕様

### 14.1 単体テスト対象

`validation.ts`

- 必須項目が空の場合にエラーを返す。
- 性別が空でもエラーにならない。
- 名前が50文字以内なら通る。
- 名前が51文字以上ならエラーになる。
- 相談テーマと今月の気分が255文字以内なら通る。
- 相談テーマと今月の気分が256文字以上ならエラーになる。

`prophecyGenerator.ts`

- 常に4週分を返す。
- 各週が4行を持つ。
- `weekNumber` が1から4になる。
- 各行が空文字ではない。
- 禁止語彙が含まれにくいことを確認する。

`promptBuilder.ts`

- ユーザー入力を含む。
- 4週分の予言を含む。
- 出力条件を含む。
- 専門判断が必要な領域では専門家相談を促す文言を含む。

### 14.2 画面テスト観点

- 必須項目未入力で生成ボタンを押すとエラーが表示される。
- 必須項目入力後に生成すると結果が表示される。
- 結果には第1週から第4週まで表示される。
- 各週の詩が4行表示される。
- 再生成ボタンで結果が更新される。
- AI用プロンプトが表示される。
- コピーボタン押下で成功または失敗メッセージが表示される。

### 14.3 レスポンシブ確認

確認幅:

- 375px
- 768px
- 1280px

確認内容:

- 入力欄が画面外にはみ出さない。
- ボタンテキストがはみ出さない。
- 予言文が読みやすい。
- AI用プロンプト欄が横スクロールを発生させない。

## 15. 受け入れ条件

初期実装は以下を満たしたら完了とする。

- 必須入力をすべて入れると予言を生成できる。
- 必須入力が欠けると該当エラーが表示され、生成されない。
- 生成結果が常に4週分である。
- 各週が常に4行である。
- 再生成で表現が変わる場合がある。
- AI用プロンプトにユーザー入力、4週分の予言、出力条件が含まれる。
- AI用プロンプトをコピーできる。
- コピー失敗時も手動コピーできる。
- 注意書きが画面内に表示される。
- 公式素材を使っていない。
- スマートフォン幅でレイアウトが崩れない。
