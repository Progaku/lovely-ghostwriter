# Issue テンプレート

```md
## 背景

<!-- 要件・詳細設計・既存実装のどこに根拠があるかを書く。 -->

## やること

- [ ] 

## 受け入れ条件

- [ ] 

## テスト観点

- [ ] 

## 対象外

- 

## 参照

- `AGENTS.md`
- `docs/**.md`
```

## 書き方

- タイトルは `種別: 内容` の形を基本にする。
  - 例: `実装: 予言入力フォームを作成する`
  - 例: `テスト: 予言生成ロジックの主要ケースを検証する`
  - 例: `設計: 初期実装の issue 分割を確定する`
- 「やること」は実装タスク、「受け入れ条件」は観測可能な完了状態にする。
- 「テスト観点」は自動テストと手動確認を分けて書く。
- 「対象外」には、初期スコープ外や次 issue に送る内容を書く。
- 「参照」は関係する文書だけに絞る。必要なら節名も書く。

## ラベル候補

既存運用がない場合は、ラベルは無理に付けない。候補を書く場合は以下を使う。

- `type: feature`
- `type: test`
- `type: docs`
- `type: chore`
- `area: ui`
- `area: generation`
- `area: prompt`
- `area: safety`
